# 项目部署文档

## 1. 环境要求 (Prerequisites)

在部署本项目之前，请确保服务器满足以下环境要求：

*   **操作系统**: Linux (Recommended) / MacOS / Windows
*   **Java 环境**: JDK 17
*   **Python 环境**: Python 3.10+
*   **Node.js 环境**: Node.js 18+ (建议使用 pnpm 管理依赖)
*   **数据库**: MySQL 8.0+
*   **缓存**: Redis 5.0+

## 2. 数据库初始化 (Database Setup)

1.  **创建数据库**:
    建议创建一个名为 `GY-EP_local` 的数据库（与 Java 默认配置保持一致）。
    ```sql
    CREATE DATABASE `GY-EP_local` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
    ```

2.  **导入数据**:
    请执行 SQL 脚本 `gz-springboot-base-dev/sql/GY-EP_local_20250418.sql` 进行数据初始化。该脚本包含了最新的表结构和基础数据。

## 3. 服务配置 (Configuration)

### 3.1 Java 后端配置
配置文件路径: `gz-springboot-base-dev/gz-server/src/main/resources/application-dev.yml` (开发环境) 或 `application-prod.yml` (生产环境)

请修改以下关键配置项：
*   **数据库连接**:
    ```yaml
    spring:
      datasource:
        hikari:
          master:
            url: jdbc:mysql://localhost:3306/GY-EP_local?...  # 修改 IP 和数据库名
            username: root  # 修改为你的数据库账号
            password: password # 修改为你的数据库密码
    ```
*   **Redis 连接**:
    ```yaml
    spring:
      redis:
        host: localhost # Redis IP
        port: 6379
        password: password # Redis 密码
    ```

### 3.2 Python 后端配置
配置文件路径: `gz-python-dev/.env.dev` (或复制一份命名为 `.env.prod`)

请确保以下配置与你的环境一致：
```ini
# 数据库配置 (注意：请确认与 Java 后端使用同一个数据库，通常应改为 GY-EP_local)
DB_HOST = '127.0.0.1'
DB_PORT = 3306
DB_USERNAME = 'root'
DB_PASSWORD = 'password'
DB_DATABASE = 'GY-EP_local'  # 建议统一修改为 GY-EP_local

# Redis 配置
REDIS_HOST = '127.0.0.1'
REDIS_PASSWORD = 'password'
```

### 3.3 Vue 前端配置
如果需要修改后端 API 地址，请检查 `gz-vue-base-dev/.env.development` 或 `.env.production` 文件中的 `VITE_APP_BASE_API`。

## 4. 启动与部署 (Deployment Steps)

### 步骤 1: 启动 Python 后端
```bash
cd gz-python-dev

# 1. 创建并激活虚拟环境 (可选，建议)
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# 2. 安装依赖
pip install -r requirements.txt

# 3. 启动服务
# 开发模式:
python app.py
# 生产模式建议使用 gunicorn (Linux):
# gunicorn -w 4 -k uvicorn.workers.UvicornWorker app:app -b 0.0.0.0:9099 --daemon
```

### 步骤 2: 启动 Java 后端
```bash
cd gz-springboot-base-dev

# 1. 编译打包
mvn clean package -Dmaven.test.skip=true

# 2. 单包发布（推荐）
cd ..
mkdir -p runtime
cp -f gz-springboot-base-dev/gz-server/target/gz-server.jar runtime/gz-server-current.jar

# 3. 启动服务（固定只运行这一份包）
nohup java -jar runtime/gz-server-current.jar --spring.profiles.active=dev > gz-springboot-base-dev/java_server.log 2>&1 &
```

### 步骤 3: 部署 Vue 前端
```bash
cd gz-vue-base-dev

# 1. 安装依赖
pnpm install

# 2. 构建生产环境包
pnpm run build:prod

# 3. 部署
# 构建完成后会生成 dist 目录，请将 dist 目录下的文件部署到 Nginx 或其他 Web 服务器。
```

## 5. 常用脚本 (Scripts)

项目根目录下提供了便捷脚本（仅限 Mac/Linux）：

*   **启动所有服务**: `./start_all.sh`
    *   该脚本会清理端口占用，并依次启动 Python、Java 和 Vue 服务（开发模式）。

---
**注意**: 部署前请务必检查 `gz-python-dev` 和 `gz-springboot-base-dev` 中的数据库配置是否指向同一个数据库实例，以免数据不一致。
