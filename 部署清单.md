### 1.2 供应商信息表初始化 ⭐ **必须执行**
**文件路径**: `gz-springboot-base-dev/sql/supplier_init.sql`  
**执行顺序**: 第 2 个执行  
**说明**: 创建供应商信息管理表并插入示例数据

#### 功能说明：
- ✅ 创建 `supplier_info` 表（如已存在会先删除）
- ✅ 插入 3 条示例供应商数据（格兰富、威乐、赛莱默）

#### 表结构关键字段：
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `id` | BIGINT | 主键ID（自增） |
| `supplier_name` | VARCHAR(200) | 供应商名称（必填） |
| `category_level1_id` | BIGINT | 一级分类ID |
| `category_level2_id` | BIGINT | 二级分类ID |
| `category_level3_id` | BIGINT | 三级分类ID |
| `brand_name` | VARCHAR(100) | 供应品牌 |
| `province/city/district` | VARCHAR(50) | 省市区 |
| `contact_person` | VARCHAR(50) | 联系人（必填） |
| `contact_phone` | VARCHAR(20) | 联系电话（必填） |
| `status` | CHAR(1) | 状态（1正常 0停用） |
| `del_flag` | CHAR(1) | 删除标识（0正常 1删除） |

#### 执行方式：
```bash
mysql -u [用户名] -p [数据库名] < gz-springboot-base-dev/sql/supplier_init.sql
```

#### 注意事项：
- ⚠️ **该脚本会先删除 `supplier_info` 表**，如果生产环境已有数据，请谨慎执行
- ⚠️ 建议在生产环境执行前，先备份现有数据
- ⚠️ 示例数据仅供测试使用，生产环境可根据需要删除或修改

## ⚠️ 六、风险提示

### 高风险操作：
1. **`supplier_init.sql` 会删除现有表**
   - 如果生产环境已有 `supplier_info` 表和数据，执行前务必备份
   - 建议修改脚本，去掉 `DROP TABLE` 语句，改用 `CREATE TABLE IF NOT EXISTS`

2. **菜单补丁会强制更新首页组件**
   - 第 31-33 行会修改首页驾驶舱的 `component` 字段
   - 如果不希望修改首页，请注释掉这部分 SQL

3. **✅ 菜单结构已修正**
   - SQL 脚本已采用方案 A 修正，菜单结构与前端路由完全一致
   - 供应商查询：单级菜单
   - 材料标准：两级菜单（与前端 children 配置匹配）
   - 无需额外调整，可直接执行

### 中风险操作：
1. **新增菜单可能影响现有权限配置**
   - 建议在测试环境先验证菜单权限分配
   - 确认各角色的菜单访问权限是否符合预期

### 低风险操作：
1. **前端代码更新**
   - 主要是新增功能，对现有功能影响较小
   - 建议在测试环境先验证功能完整性
